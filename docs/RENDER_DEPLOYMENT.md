## MongoDB Setup

### Step 1: Create MongoDB Atlas Cluster

1. Go to https://www.mongodb.com/cloud/atlas
2. Sign in or create a free account
3. Create a new project (e.g., "AIRO-Prod")
4. Create a new cluster:
   - Select **Free Tier** for development
   - Choose region closest to your app
   - Create cluster and wait for it to initialize

### Step 2: Database and User Setup

1. In MongoDB Atlas, go to **Database Access**
   - Create a new database user
   - Username: `airo_user`
   - Password: Generate strong password (copy it!)
   - Add user

2. Go to **Network Access**
   - Add IP Address: **0.0.0.0/0** (allows all IPs - secure with Auth)
   - Confirm

3. Go to **Clusters** → **Connect**
   - Choose "Connect your application"
   - Copy the connection string
   - Replace `<password>` with your database password
   - Replace `myFirstDatabase` with `airo_db`
   
   **Example:**
   ```
   mongodb+srv://airo_user:YOUR_PASSWORD@cluster.mongodb.net/airo_db?retryWrites=true&w=majority
   ```

## Render Dashboard Setup

### Step 1: Create Backend Service

1. Go to https://dashboard.render.com
2. Click **New +** → **Web Service**
3. Connect your GitHub repository
4. Fill in details:
   - **Name:** `airo-backend`
   - **Root Directory:** `backend`
   - **Environment:** Docker
   - **Plan:** Starter ($7/month - required for ML models)
   - **Region:** (choose closest to you)

5. Build & Deploy
   - Build Command: (leave empty - uses Dockerfile)
   - Start Command: (leave empty - uses Procfile)

6. Add Environment Variables:
   - Click **Environment**
   - Add these variables (copy from `backend/.env.example`):

| Key | Value | Notes |
|-----|-------|-------|
| `MONGO_URI` | `mongodb+srv://airo_user:PASSWORD@cluster.mongodb.net/airo_db?retryWrites=true&w=majority` | Replace PASSWORD |
| `DB_NAME` | `airo_db` | - |
| `JWT_SECRET_KEY` | (Auto-generated) | Render generates this securely |
| `JWT_ALGO` | `HS256` | - |
| `GROQ_API_KEY` | Your Groq API key | Get from https://console.groq.com |
| `GITHUB_TOKEN` | (Optional) | Your GitHub token |
| `MAIL_USERNAME` | your-email@gmail.com | Gmail address |
| `MAIL_PASSWORD` | Your Gmail App Password | https://myaccount.google.com/apppasswords |
| `MAIL_FROM` | noreply@airo.com | Sender email |
| `MAIL_SERVER` | smtp.gmail.com | - |
| `MAIL_PORT` | 587 | - |
| `MAIL_FROM_NAME` | AIRO Support | - |
| `FRONTEND_URL` | (Set after frontend deployed) | e.g., https://airo-frontend-xxx.onrender.com |

7. Deploy
   - Render will automatically deploy when you save
   - Wait for build to complete (5-10 minutes for first deployment)
   - Note the backend URL: `https://airo-backend-xxx.onrender.com`

### Step 2: Create Frontend Service

1. Click **New +** → **Web Service** (again)
2. Select the same GitHub repository
3. Fill in details:
   - **Name:** `airo-frontend`
   - **Root Directory:** `frontend`
   - **Environment:** Node
   - **Plan:** Starter
   - **Region:** (same as backend)

4. Build & Deploy Commands
   - Build Command: `npm ci && npm run build`
   - Start Command: `npm start`

5. Add Environment Variables:

| Key | Value |
|-----|-------|
| `NEXT_PUBLIC_API_URL` | `https://airo-backend-xxx.onrender.com` |
| `NODE_ENV` | `production` |
| `NODE_OPTIONS` | `--max-old-space-size=1024` |

6. Deploy
   - Wait for build (3-5 minutes)
   - Note the frontend URL: `https://airo-frontend-xxx.onrender.com`

### Step 3: Update Backend FRONTEND_URL

1. Go back to backend service
2. Go to **Environment**
3. Add/Update `FRONTEND_URL`:
   - Value: `https://airo-frontend-xxx.onrender.com`
4. Save & redeploy

## Environment Configuration

### Backend Environment Variables

All backend variables should be set in Render Dashboard, **NOT** in `.env` files.

**Critical variables:**

- **`MONGO_URI`** - Database connection string (from MongoDB Atlas)
- **`JWT_SECRET_KEY`** - Auto-generated by Render (or generate: `python -c "import secrets; print(secrets.token_urlsafe(32))"`)
- **`GROQ_API_KEY`** - AI API key from https://console.groq.com
- **`MAIL_PASSWORD`** - Gmail App Password (not your Gmail password!)

### Frontend Environment Variables

- **`NEXT_PUBLIC_API_URL`** - Backend API URL (auto-prefixed with `NEXT_PUBLIC_`)
- **`NODE_ENV`** - Must be `production`
- **`NODE_OPTIONS`** - Memory management for build process

## Troubleshooting

### Backend won't start

**Error:** Service dies after deployment

**Solutions:**
1. Check environment variables are set (especially `MONGO_URI`)
2. Verify MongoDB connection string is correct
3. Check Render logs for specific errors
4. Ensure Dockerfile and Procfile are correct
5. Verify Python dependencies in requirements.txt

**Command to check logs:**
```bash
# In Render Dashboard, view service logs
```

### Frontend won't build

**Error:** Build fails with memory error

**Solution:**
- Increase `NODE_OPTIONS: --max-old-space-size=2048`
- Upgrade to Professional plan if needed
- Optimize large dependencies

### CORS errors

**Error:** Frontend can't reach backend API

**Solutions:**
1. Verify `FRONTEND_URL` is set in backend env
2. Verify `NEXT_PUBLIC_API_URL` is set in frontend env
3. Check both URLs are correct (https://, not http://)
4. Wait 2-3 minutes after deployment for propagation

### Database connection timeout

**Error:** Can't connect to MongoDB

**Solutions:**
1. Verify MongoDB cluster is running
2. Check IP whitelist in MongoDB Atlas (should include 0.0.0.0/0)
3. Verify `MONGO_URI` has correct password
4. Ensure database name is correct

### Slow builds or deployments

**Reason:** Free tier limitations

**Solution:**
- Upgrade to Starter plan ($7/month)
- Don't have heavy computations on frontend build
- Frontend size should be < 500MB

## Performance Tips

1. **Backend**
   - ML models pre-download in Docker (already done)
   - Keep Starter plan for adequate RAM
   - Monitor spacy and sentence-transformer model sizes

2. **Frontend**
   - Uses `standalone` output (optimized for deployment)
   - Next.js auto-optimizes images
   - Consider adding CDN for assets

3. **Database**
   - Keep MongoDB Atlas on free tier for development
   - Create indexes on frequently queried fields
   - Backup regularly (MongoDB Atlas has auto-backups)
